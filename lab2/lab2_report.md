# Отчет по лабораторной работе
## Настройка CI/CD пайплайна с GitHub Actions

## Цель работы
Целью лабораторной работы является настройка CI/CD пайплайна с использованием GitHub Actions для автоматической сборки Docker-образа приложения и его публикации в Docker Hub при каждом обновлении кода в репозитории.

---

## Подготовка проекта
Для выполнения лабораторной работы был создан новый репозиторий на GitHub.  
В репозиторий были добавлены файлы из предыдущей лабораторной работы:

- `app.py` — основной файл Flask-приложения
- `requirements.txt` — файл с зависимостями Python
- `Dockerfile` — файл для сборки Docker-образа

Также было проведено тестирование запуска docker c Flask приложением.

## Настройка GitHub Actions
Для автоматизации сборки и публикации Docker-образа был настроен CI/CD пайплайн с использованием GitHub Actions.

В проекте была настроена работа автоматической сборки и отправки образа в docker hub с испольованием следующих 
параметров:
Данный файл описывает процесс выполнения пайплайна.

### Основные этапы пайплайна

1. Запуск пайплайна при каждом `push` в ветку `main`
2. Использование `ubuntu-latest` в качестве runner
3. Получение исходного кода из репозитория (checkout)
4. Настройка Docker Buildx
5. Авторизация в Docker Hub
6. Сборка Docker-образа
7. Публикация Docker-образа в Docker Hub


## Секреты
Для безопасного хранения данных авторизации Docker Hub в настройках GitHub-репозитория были добавлены секреты:

DOCKER_USERNAME — логин Docker Hub

DOCKER_PASSWORD — пароль или access token Docker Hub

Эти секреты используются в workflow для выполнения авторизации в Docker Hub.


## Тестирование

выполнение в разделе "Actions" репозитория и наличие образа в docker Hub. Подтверждения приложены в виде скриншотов
